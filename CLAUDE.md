## High‑Level Architecture

- **Framework:** Astro 6 (SSR) – pages live under `src/pages`. Dynamic routes are expressed with bracket syntax (`[param]`). The project outputs **static** assets (`output: 'static'`) but runs as a Node server in development via the `@astrojs/node` adapter.
- **Styling:** Tailwind CSS v4 with the official Tailwind Vite plugin. Global styles are imported in `src/layouts/Layout.astro`.
- **Content Rendering:**
  - MDX support (`@astrojs/mdx`) for rich markdown content.
  - LaTeX math rendered by KaTeX (`rehype-katex` + client‑side KaTeX script).
  - Markdown content is enhanced by `remark-math`.
- **Data Layer:** Supabase is the sole backend service.
  - Auth is handled via `@supabase/supabase-js`.
  - In **Mock Mode** (default for a fresh checkout) the app falls back to hard‑coded demo accounts and does not require a real Supabase project.
  - Production mode reads connection strings from a `.env` file (`SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`).
- **Core UI Components** (found under `src/components`):
  - `Layout.astro` – global page wrapper handling SEO, Open Graph, JSON‑LD, dark‑mode handling, and optional AdSense.
  - `Navbar`, `Sidebar`, `Footer` – navigation and site chrome.
  - `ChartComponent` – wraps Chart.js for dashboards.
  - `AIChat` – client‑side chat UI that talks to a Gemini‑style LLM endpoint (key read from `GEMINI_API_KEY`).
- **Feature Areas (pages)**
  - **Student portal** (`src/pages/*.astro`): home, lessons (`/khoa-hoc/[slug]`), theory (`/ly-thuyet/[slug]`), exams (`/exams/[id]`), AI chat (`/ai-chat`).
  - **Admin portal** (`src/pages/admin/*`): user management, questions bank, blog posts, courses, lessons, exams. All admin routes are guarded by Supabase auth roles.
  - **Static assets** (`public/`): favicons, images.
- **State Management** – relies primarily on Astro’s server‑side rendering and the `Astro.locals.user` object populated in each request via Supabase auth middleware.
- **Analytics / Ads** – optional Google AdSense script is conditionally injected based on the `showAds` prop.
- **Deployment adapters** (described in README, also reflected in `astro.config.mjs` when swapped):
  - Cloudflare Pages (`@astrojs/cloudflare`)
  - Vercel (`@astrojs/vercel`)
  - Netlify (`@astrojs/netlify`)

## Important README Highlights

- The project targets **Node ≥ 22.12** and uses TypeScript 6.
- **Mock Mode** enables offline development with three demo accounts:
  - Student – `student@lop12.vn`
  - Teacher – `teacher@lop12.vn`
  - Admin – `admin@lop12.vn`
- Production setup requires applying the two migration files located in `supabase/migrations/`.
- Deployment steps are enumerated for Cloudflare, Vercel, and Netlify; they mainly involve installing the corresponding Astro adapter and pushing the repo to GitHub.
